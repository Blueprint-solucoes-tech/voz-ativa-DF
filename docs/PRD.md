# PRD — <Nome do Projeto>

## 1. Contexto
Descreva o cenário atual.
Por que esse projeto existe agora?

## 2. Problema
Qual dor real está sendo resolvida?
Quem sente esse problema?

## 3. Objetivo
O que significa “sucesso” para este projeto?
Frase curta e clara.

## 4. Usuários
Quem vai usar?
Liste perfis principais.

Ex:
- Usuário final
- Admin
- Visitante

## 5. Casos de uso principais
O que o usuário consegue fazer?

Ex:
- Visualizar X
- Criar Y
- Filtrar Z

## 6. Requisitos funcionais
O sistema DEVE fazer.

- [ ] Requisito 1
- [ ] Requisito 2

## 7. Requisitos não funcionais
Qualidades do sistema.

Ex:
- Acessibilidade (WCAG AA)
- Performance
- Segurança
- Usabilidade

## 8. Fora de escopo (Non-goals)
O que NÃO será feito agora (importante!).

- ❌ Exemplo

## 9. Premissas e restrições
O que estamos assumindo como verdade?

Ex:
- Prazo curto
- Time pequeno
- Stack fixa

## 10. Critérios de sucesso
Como saberemos que deu certo?

Ex:
- Usuário consegue fazer X sem ajuda
- Tempo de carregamento < 2s
